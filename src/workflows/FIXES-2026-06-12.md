# Workflow Fixes - June 12, 2026

## Issues Found

1. **Disk Space Crisis**: 8.5GB unified-report.txt filled disk to 82% capacity
2. **QA Server URL Mismatch**: Script used localhost:8884 instead of remote server
3. **Missing Basic Auth**: QA server requires authentication that wasn't being passed
4. **Unbounded Report Generation**: No limits on SF report output size

## Root Cause

The QA workflow failed because:
1. QA tests couldn't connect to the server (wrong URL, missing auth)
2. Setup test failed with "Cannot navigate to invalid URL"
3. All 14 tests were skipped due to setup failure
4. Report generation got stuck and kept writing output until disk was full

## Fixes Applied

### 1. report-builder.sh
- Added 10,000 line limit for SF reports to prevent disk issues
- Added file size display in report header
- Excluded unified-report.txt from SF report search (prevent recursive include)
- Added truncation notice when reports are cut off for safety

### 2. qa-client.sh
- Changed QA_SERVER_URL to use QA_URL from .env (http://50.28.85.146:8888)
- Added basic auth support using QA_USER and QA_PASS from .env
- Fixed check_qa_server() to include auth credentials
- Used proper bash arrays for curl commands (shellcheck compliant)
